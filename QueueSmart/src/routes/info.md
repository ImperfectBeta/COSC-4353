`src/routes/(marketing)` is where "front-facing" pages should go.
This includes things like the landing page (think of the page that advertises QueueSmart to potential customers) and things like that.
`src/routes/(marketing)/+layout.svelte` is the layout for the marketing pages.

`src/routes/(app)` is where the actual application pages should go.
This includes things like the dashboard, the queue management pages, etc.
`src/routes/(app)/+layout.svelte` is the layout for the application pages.

`src/routes/(auth)` is where authentication pages should go.
This includes things like the login page, the signup page, etc.
`src/routes/(auth)/+layout.svelte` is the layout for the authentication pages.
